# RuTalk

RuTalk — это веб-приложение социальной сети на Django, где пользователи могут объединяться в **группы** и **каналы**, публиковать посты, комментировать и общаться по интересам.

Проект реализован в рамках НИС «Введение в облачные технологии».  
Репозиторий содержит код, документацию и примеры для локального развертывания.

---

## 🚀 Основные возможности

- Регистрация, аутентификация, смена и сброс пароля (встроенные views Django + кастомные шаблоны)
- Редактируемый профиль с дополнительными полями (имя, фамилия, отчество, дата рождения, био, место учебы/работы, телефон, аватар)
- Подписки на других пользователей
- **Группы** – любой участник может создавать посты
- **Каналы** – посты создает только владелец, участники могут читать и комментировать
- Посты с текстом и изображениями (jpg, jpeg, png, gif, до 10 МБ)
- Комментарии к постам (тоже с возможностью загрузки изображений)
- Поддержка облачного хранилища Yandex Object Storage (S3) через django-storages

## 🗃 Модели данных

- **User** – стандартная модель Django
- **Profile** – расширение пользователя (avatar, имя, фамилия, отчество, bio, birth_date, education, phone, follows)
- **Group** – тематическая группа (name, description, owner, created_at)
- **Channel** – канал (name, description, owner, created_at)
- **Membership** – связь пользователя с группой (unique_together)
- **ChannelMembership** – связь пользователя с каналом (unique_together)
- **Post** – пост (user, group или channel, body, image, created_at)
- **Comment** – комментарий (user, post, body, image, created_at)

### Особенности связей
- Профиль автоматически создается при регистрации (сигнал `post_save`) и подписывается на самого себя.
- Пост должен принадлежать **ровно одному** контейнеру (группе или каналу) – проверка в `save()`.
- Удаление пользователя (`DO_NOTHING` в полях `user` моделей `Post` и `Comment`) требует дополнительного контроля целостности (установлено вручную).
